# v1.5 — Gate sweep (the 9 soft-gating configs): formulas, configs, result, how to improve

> **TL;DR.** We swept 9 configs of an **end-to-end soft do-no-harm gate** on the
> mem-X wrapper. **All 9 failed** (the gate sits at $g\approx0.5$ and the per-token
> residual corrupts generation). The negative result motivated the **hard routing
> gate** that does work (see [`v1.5-metric-candidates-2026-06-04.md §7`](../2026-06-04/v1.5-metric-candidates-2026-06-04.md) and the matrix §7).

Code: `mem-embedding/scripts/gate_train.py`, `src/mem_embedding/wrapper.py` (`combine='gated'`). ## 1. What a "gated combine" is (the formula)

The wrapper reads memory into the frozen base by a **convex residual** modulated
by a scalar gate $g\in(0,1)$:

$$ e'_t \;=\; e_t \;+\; g\cdot \alpha\cdot \Delta e_t, \qquad \Delta e_t=\mathrm{LN}\big(\mathrm{CrossAttn}(e_t,\,m)\big) $$

- $e_t$ = the base token embedding; $\Delta e_t$ = the memory's per-token contribution; $\alpha$ = a learned scale.
- **Do-no-harm by construction:** $g\to 0 \Rightarrow e'_t=e_t$ (the frozen base is untouched). Verified exactly in a unit test (max$|e'-e|=0$ at $g{=}0$).

The gate is an MLP on **intrinsic memory-write features** (optionally query-conditioned):

$$ g \;=\; \sigma\big(\mathrm{MLP}(\phi)\big),\qquad \phi=\big[\underbrace{\overline{\|m_k\|},\,\max_k\|m_k\|,\,\|m_T-m_0\|,\,\|\Delta m_T\|}_{\text{memory-write (model-general)}}\,[,\ \underbrace{\bar q,\ \cos(\bar q,\bar m),\ \max_k\cos(\bar q,m_k)}_{\text{query-conditioned}}\,]\big] $$

`g_bias_init` sets the gate-output logit bias: $0\Rightarrow g{\approx}0.5$; $-2\Rightarrow g{\approx}0.12$ (start near-closed).

## 2. Training objective (`gate_train.py`)

Per step we sample a **real** item and build a **decoy** = the real item's query
but memory built from a *different* item's chunks (foreign / irrelevant context):

$$ \mathcal{L} \;=\; \underbrace{\mathrm{CE}_{\text{real}}(\text{gold span})}_{\text{task}} \;+\; \lambda_{\text{dnh}}\,\underbrace{\mathrm{KL}\big(p_{\text{gated}}^{\text{decoy}}\,\big\|\,p_{\text{no-ctx}}\big)}_{\text{do-no-harm on decoy}} \;+\; \lambda_{\text{gate}}\,\underbrace{\big[\mathrm{BCE}(g_{\text{real}},1)+\mathrm{BCE}(g_{\text{decoy}},0)\big]}_{\text{gate target}} \;+\; \text{ans-head aux} $$

Intuition: open the gate where memory is relevant (real → $g\to1$), close it where
it is not (foreign decoy → $g\to0$), and on decoys force the gated output to match
the no-context base (do-no-harm).

## 4. Result — all 9 FAIL (the honest negative)

- **Gate doesn't separate:** $g_{\text{real}}\approx g_{\text{decoy}}\approx0.45$–$0.50$ throughout training (even at $\lambda_{\text{gate}}{=}5$, `mem`-only c5 identical to query variants early).
- **Eval is worse than ungated:** on the finished configs, gated trivia $\to 0.00$ (vs no-context $0.485$), hotpot $\to 0.00$ — the per-token residual at $g{\approx}0.5$ **corrupts generation**.
- The `g_bias_init` knob *does* set the starting gate ($-2\Rightarrow0.12$, $-1\Rightarrow0.27$), confirming the wiring; the gate just doesn't *learn to discriminate*.

**Why it fails (root causes):**
1. **Foreign-context decoys don't separate via the gate features.** A foreign memory has *similar* write-magnitude to a real one, and the query↔memory alignment signal is weak/slow to learn — so the gate can't tell "relevant" from "irrelevant."
2. **cat_niah-only training has no regime signal.** The wrapper's real failure mode is *detail-needed* tasks (Regime B), but cat_niah is all compressible (Regime A) — the gate never sees a "should-close" example of the kind that matters at eval.
3. **The per-token residual corrupts generation** at any non-trivial $g$ (the do-no-harm property is only *exact* at $g{=}0$).

## 5. How to improve (ranked)

1. **Hard routing instead of soft mixing (✅ done, works).** Route base$\leftrightarrow$wrapper by $g=P(\text{useful})$ at a threshold $\tau$ — both endpoints are "clean," so no residual corruption. This is the gate we ship (do-no-harm 32/35; LOMO AUROC 0.71). See `gate_route.py`.
2. **Supervise the gate on real usefulness labels** (the E8 logistic on the 7-model grid) rather than the cat_niah decoy proxy — gives a calibrated $P(\text{useful})$ that transfers.
3. **Regime-spanning training data**: mix compressible (cat_niah) *and* verbatim/detail (ruler-style) so the gate learns the A/B boundary that actually predicts harm.
4. **Gate the prefix/`xattn` read, not the per-token residual** — fewer degrees of freedom to corrupt generation; do-no-harm becomes "drop the memory prefix."
5. **Make the gate respond from step 0**: non-zero gate-head init + a *contrastive* (real vs decoy) margin loss instead of independent BCE; harder, in-distribution decoys.
6. **Temperature-calibrated threshold** on the routing gate (reliability diagram) to set $\tau$ per the do-no-harm tolerance.
