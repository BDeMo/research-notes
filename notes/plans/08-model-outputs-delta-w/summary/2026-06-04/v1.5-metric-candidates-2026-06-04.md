## 1 · Rubric (4-axis initial screen)

Each candidate is rated ★–★★★ on the user's four screens; verdict combines them.

- **泛用性 Generality** — does the signal transfer across base families/tasks? (★★★ = wrapper-/structure-intrinsic, family-agnostic; ★ = reads one family's idiosyncratic geometry). *Data-driven*: the 2-model run already shows logit-lens read-outs are family-specific.
- **没人做过 Novelty** — ★★★ = not used as a gate/harm signal anywhere; ★ = standard (e.g. raw output confidence / selective prediction).
- **有趣 Interesting** — ★★★ = non-obvious, teaches mechanism after analysis; ★ = expected/tautological.
- **合理 Soundness** — ★★★ = principled, hard-to-game, well-defined; ★ = ad-hoc / estimator-noisy.
- **Codeable** — every KEEP must be implementable in `probe_v2.py`'s per-item loop (pass + #extra forwards noted). Verdict: 🟢 CODE&TEST · 🟡 SUPPORT/extension · 🔴 drop (or keep only as a family-specific diagnostic).

## 4 · Shortlist to validate first (cheap × general × novel × interesting)

Order = expected information per GPU-minute, breaking ties toward novelty + the 09 bridge.

1. **R1 `mem_attn_mass` + R2 `mem_read_by_retrieval_frac`** — attention-routing gate; most novel + the cleanest 08↔09 bridge. (eager attention + 09 retrieval-head detector.)
2. **C1 `mem_input_specificity`** — shuffled-context counterfactual; 1 extra forward, very novel, directly tests "is the wrapper reading THIS input."
3. **M1 `mem_eff_rank` + M2 `q_chunk_align_entropy`** — wrapper-intrinsic, ~free, maximally general.
4. **S1 `mem_massive_act_perturb`** — 09 bridge; "is the wrapper hitting the base's load-bearing channels."
5. **C2 `mem_dose_curvature` + U1 `mem_dropout_var`** — counterfactual stability; a few extra forwards, high wow-factor if they predict harm.

## 5 · Validation protocol (the "one-by-one" experiment)

For each candidate, add it to `probe_v2.py`'s per-item record, then on the
**3-model cohort** (Qwen3-8B / GLM-4-9B / Qwen3.5-9B) score it with
`xmodel_consistency.py`. A candidate **passes** iff:

1. **Predictive** — `|AUROC(help) − .5| ≥ 0.08` and/or `|AUROC(no-harm) − .5| ≥ 0.08` pooled.
2. **General** — same direction in **all 3** families (the screen that killed `top5_mass_w`).
3. **Non-redundant** — `|ρ|` with an already-KEPT signal `< 0.8` (drop mechanical twins, à la 09).
4. **Adds at the gate** — improves a small logistic gate's held-out AUROC over the current memory-geometry + base-difficulty baseline.

Survivors graduate into the gate feature set (Phase 4) and the codebook in the v1.5 tex.

## 7 · RESULTS — 7-model verdicts (2026-06-05)

Full grid run with `probe_v3` (all candidate metrics) on **7 model families/scales**
(Qwen3-8B, Qwen3-14B, Qwen3.5-9B, Qwen2.5-7B, GLM-4-9B, Phi-3.5-mini, Mistral-7B),
5 core benches, 13.2k items. Grid: [`grids-xmodel-2026-06-05/xmodel_consistency_7models.csv`](../../raw/grids-xmodel-2026-06-05/xmodel_consistency_7models.csv) (+ `.png`).

**Direction-consistent across all 7 (13/90 comparable signals).** The robust,
gate-worthy set is **wrapper-intrinsic memory-write dynamics** (all ↑ "bigger
write ⇒ more likely to help"), strongest on the non-Qwen families:

| signal | angle | range AUROC(help) | note |
|---|---|---|---|
| **`delta_last`** | geometry | 0.59–0.80 | **only signal clearing |AUROC−.5|≥.07 in *all* 7** |
| `slot_norm_max` / `slot_norm_mean` | geometry | 0.51–0.83 | memory magnitude |
| `delta_mean` / `delta_trend` / `drift_mem` | geometry | 0.52–0.82 | update dynamics |
| **`mem_influence_span`** (NEW, C3) | causal | 0.52–0.70 | the one new candidate that **generalizes** |
| `top5_mass_w`, `conf_w` | confidence | dir-consistent but **weak on GLM/Qwen3-14B (~0.51)** | partial |

**New-candidate verdicts (of the 15):**
- ✅ **GENERAL — keep for the gate:** `mem_influence_span` (causal; 7/7 same dir).
- ⚠️ **MODEL-SPECIFIC** (strong on some, flips on others — useful per-model, not a universal gate input): `mem_anisotropy` (0.83 @Phi), `span_js_w0` (0.75 @Qwen3-14B), `mem_attn_mass` (0.66 @Phi, 0.60 @GLM), `mem_massive_act_perturb` (0.64 @Qwen3-14B), `mem_dose_curvature` (0.60 @GLM), `mem_input_specificity` (0.59 @Qwen3-14B).
- ❌ **WEAK / noise:** `mem_eff_rank`, `q_chunk_align_entropy`, `slot_gini`, `hidden_effrank_ratio_w0`, `mem_dropout_*`, `mem_read_by_retrieval_frac`, `mem_attn_layer_profile`.

**Mechanistic reading (the paper sentence).** Signals that are properties of the
**wrapper itself** (memory-write magnitude/dynamics) or of the wrapper's **causal
effect on the answer span** (`mem_influence_span`) transfer across base families;
signals that read the **base model's internal geometry** (logit-lens conf/margin)
or family-specific behaviours are **model-specific**. ⇒ Build the do-no-harm gate
on the memory-geometry family + `mem_influence_span`; treat logit-lens/attention
signals as per-model features at most.

**E8 — gate ceiling + cross-family transfer** ([`grids-xmodel-2026-06-05/gate_*.csv`](../../raw/grids-xmodel-2026-06-05/)).
Logistic gate predicting `useful` (lift>0): pooled 5-fold CV AUROC 0.55 (`delta_last`)
→ 0.62 (geom+span+conf) → 0.68 (all signals, but that's within-distribution).
**Leave-one-model-out transfer (predict on an UNSEEN family)**: `delta_last` alone
generalizes best — **mean AUROC 0.71 (0.63–0.82)**; the all-signal gate is unstable
across families (0.46–0.79). ⇒ a do-no-harm gate from the memory-write signal
**transfers to unseen base models** without per-model tuning.

**E5 — gated combine implemented & validated** (`wrapper.py combine='gated'`):
`e'_t = e_t + g·α·Δe_t`, `g=σ(MLP(memory-write feats))`. Unit test confirms the
**exact do-no-harm property**: `g→0` ⇒ output identical to frozen base (max|Δ|=0),
`g→1` ⇒ full wrapper. Next: train it with decoys + do-no-harm KL reg, eval vs baselines.

## 6 · Implementation notes (so it's all "coding")
- M1–M4, S2: from `m.payload` / existing hidden states — **no extra forward**.
- C1: one forward with memory built from a permuted chunk order (or a foreign item). C2: 2 forwards scaling injected memory embeds by α. C3/U2: reuse the answer-span forward already in `span_stats`.
- S1/S3: detect base massive-act channels & sink mass on the no-context pass (already computed); diff against the wrapper pass.
- R1–R3: set the base to **eager attention** and `output_attentions=True`; sum attention over the K memory key positions. R2 needs the per-model retrieval/sink head sets from plan 09's `runs/janus_*` detectors (import, don't reimplement).
- U1: N (≈8) forwards with a random slot-dropout mask on `m.payload`.
