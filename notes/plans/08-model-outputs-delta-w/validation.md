# Plan 08 — Validation

## Validation hypothesis

> *If we let an LLM output a weight delta $\Delta W$ alongside each answer and apply it under verifier control, the model improves measurably within a session — at metrics impossible to achieve with frozen weights at the same inference budget.*

## Three-phase research plan

This plan is too ambitious for a single experiment. We phase it.

### Phase 1 — Feasibility (months 1–3)

**Goal**: prove the closed loop works at all in a controlled setting.

**Setting**: synthetic continual learning task.
- A stream of (mini-task, gold) pairs, e.g., sequential SQuAD or a generated multi-step reasoning game.
- Base: Qwen2.5-7B-Instruct (or Gemma-2-2b for cheaper iteration, matching D2L).
- Hypernet $H_\phi$ initialized from a Doc-to-LoRA checkpoint.
- Output schema:
  ```
  <reasoning>...</reasoning>
  <answer>...</answer>
  <learn>Brief description of the rule / fact to remember.</learn>
  ```
  The `<learn>` text is the input to $H_\phi$, which outputs $\Delta W$.
- Verifier: rule-based (does the *next* task on the stream succeed under the new $\theta$?).

**Training the closed loop**:
- Stage 1a (warmup): supervised — pair (task-stream, oracle ΔW-trajectory generated by post-hoc context distillation). Train $H_\phi$ to match.
- Stage 1b (RL): reward = improvement on a probe set after each $\Delta W$ application; entropy bonus for diverse `<learn>` content.

**Metrics**:
- Per-application **benefit rate**: $P(\text{probe acc improves after applying} \Delta W)$
- Per-application **harm rate**: $P(\text{probe acc on held-out drops by} > \delta)$
- **Cumulative gain** over a 50-turn session.

**Pass condition**: benefit rate > 2 × harm rate; cumulative gain > 10% absolute on the in-distribution probe; cumulative drop on out-of-distribution probe < 5%.

### Phase 2 — Utility on real personalization / continual tasks

**Setting**: multi-turn personalization.
- Channel A: **LaMP** (personalization across 7 sub-tasks).
- Channel B: **PerLTQA** (long-term conversational memory).
- Channel C: **CLAM-style continual NLI** (sequential domain shift).

**Method**: same architecture as Phase 1, retrained on these task streams.

**Baselines**:
- Frozen base model (lower bound)
- RAG with the same conversation history (the standard comparison)
- D2L-as-memory: at each turn, run D2L on the conversation history to internalize. Same hypernet machinery, different trigger.
- Backprop TTT: real gradient descent per turn (oracle-ish; expensive).
- **Ours**: model emits `<learn>` → $H_\phi$ → $\Delta W$ → verifier-gated apply.

**Metrics**:
- Task-specific accuracy / F1.
- Memory-task accuracy (does the model recall facts from earlier turns?).
- Compute per turn (extra hypernet forward pass adds ~0.5s at D2L scale).

**Pass condition**: beats frozen base by ≥ 5% absolute on at least 2 of the 3 channels; competitive (within 2%) with backprop-TTT at < 10% the compute.

### Phase 3 — Agent integration

**Setting**: long-horizon agentic tasks.
- **SWE-Bench-Verified** — multi-step code repair. Self-modifying agent should improve within the task as it explores the repo.
- **WebArena / OSWorld** — multi-step GUI tasks.
- **MLE-Bench** — long ML engineering tasks.

**Method**: the agent's reasoning naturally produces `<learn>` segments after each tool-use observation; the hypernet writes accumulated repo / environment understanding into weights.

**Baselines**:
- Frozen base agent (typical setup today).
- Agent with vector-DB memory (RAG-of-experience).
- Agent with explicit summary-in-prompt memory.

**Metrics**:
- Final task success rate.
- Steps-to-solution.
- **Per-step improvement rate**: does success at step $t$ > step $t-1$?
- Total inference budget.

**Pass condition**: ≥ 5% absolute improvement in success rate over frozen agent at matched inference budget; observable within-task learning (per-step improvement rate > 0).

## Required ablations

| Ablation | Tests |
|---|---|
| **Verifier types** | rule-based / learned reward / user feedback. Which works at which task? |
| **$\alpha$ schedule** | binary apply/reject vs scaled $\alpha \in [0,1]$ vs adaptive |
| **Rollback policy** | with vs without rollback on negative reward |
| **No-`<learn>` control** | hypernet always emits ΔW from reasoning trace — does explicit `<learn>` help vs implicit |
| **Capacity-preserving constraint** | AlphaEdit-style null-space projection on $\Delta W$ — does it kill catastrophic forgetting? |
| **Forget rate** | how many turns of "$\Delta W$" before base capability degrades by > 5%? |

## Risk register

| Risk | Likelihood | Mitigation |
|---|---|---|
| Catastrophic forgetting after K updates | very high | AlphaEdit constraint; sparse memory finetuning (Lin 2025); rollback policy |
| RL doesn't converge (reward too sparse) | high | Supervised warmup (Phase 1a) is essential before RL |
| The model collapses to "always emit ΔW=0" | medium | Entropy bonus on `<learn>` content + minimum-magnitude regularizer |
| Verifier hacking — model games the verifier | medium | Out-of-distribution probe; auditing on adversarial cases |
| Phase 1 works but doesn't transfer to Phase 2 | medium | Phase 1 task design must match Phase 2's structure |
| 70B scale infeasible budget-wise | high | Keep paper-scale at 7B; cite 70B as future work |

## Theoretical questions worth answering on the way

1. What's the *capacity* of the system — how many distinct $\Delta W$ can be applied before they interfere?
2. Is there a "weight memory consolidation" sleep phase that compresses many small $\Delta W$ into one larger one?
3. Privacy: do per-session updates leak across users in a multi-tenant setting? (Almost certainly yes without explicit isolation — needs careful per-request scoping.)
4. Is this a path toward **lifelong** learning, or just longer-coherent sessions?

## Statistical practice

- All headline runs with ≥ 3 seeds per condition.
- Phase 2 + 3 should pre-register the headline comparison.
- Report **per-task** results, not just averaged ones — averaging hides the per-task variance that matters for agent applications.
