## GCM recipe (default / "best" — every main + matrix + ablation GCM cell)
`harness --methods no_ctx,full_ctx,trunc,gcm` +
```
--enc-layers half --n-memory 16 --n-dec-layers 2 --base-lora-rank 32 \
--lam-task 1.0 --lam-distill 0.5 --lam-rec 1.0 --lam-dev 0.05 \
--enc-init copy --inject prefix \
--lr 3e-4 --grad-accum 8 --epochs 4 --patience 2 --n-items 512 \
--n-eval 96 --max-ctx-tokens 1024 --signals
```
- **memory injection:** `prefix` = soft-prompt (K input-embedding vectors). *(KV-cache injection `--inject kv` = per-layer K/V, the Cartridge/Beacon/500x family; ~L× cost, reported separately.)*
- **encoder:** `copy` = trainable copy of the base's first ⌊L/2⌋ layers (`svc/compressor.py`). *(open option: `shared-lora` to reuse the frozen base + LoRA, ~½ memory — pending ablation.)*
- **distillation target (v1.7.6 PERF):** full-ctx teacher distribution over the **gold answer in ONE forward** (was token-by-token no-KV-cache gen — the precompute bottleneck).
- **patience:** held-out val loss checked every ~steps/10; stop after `patience` non-improving checks (>1e-4) = "converged". epoch cap bounds the worst case.
- **epochs↔steps:** `steps = epochs × |train set|` (1 epoch = every item once); `--steps` is the legacy fallback (epochs=0).

## Self-compressor (gcm lib) — LoRA placement + planned ablation (TO ABLATE)
The "self-compressor" design (one frozen base = its own compressor; `gcm` package) has **two stages** with distinct adapter placement:
| stage | what runs | LoRA |
|---|---|---|
| **encode** (produce M from the base's own features) | frozen base over `[ctx ; query ; K mem-tokens]` → mem-position hiddens → nonlinear `m_proj` → M | **none** (frozen-only); only `mem-tokens` + `m_proj` are trained |
| **read-M** (base reads M to answer) | base over `[M ; query]` | **read-LoRA ON** (q_proj/v_proj); disabled on the fallback/full path ⇒ exact base = do-no-harm |

So today trainable = `mem-tokens + m_proj + read-LoRA`; the **encode stage has NO LoRA**.
**Ablation to run:** add a separate, toggleable **encode-LoRA** (ON only during encode) → **frozen-only vs +encode-LoRA** (capacity vs minimalism). Both LoRAs live on the **same shared base**, toggled per stage, **never merged** (do-no-harm preserved). Implemented in `gcm` with `enc_mode ∈ {frozen, encode_lora}` once the ablation is scheduled.

## WandB recipe run (gcm self-compressor — the default recipe we're nailing)
The self-compressor is trained/tracked through the standalone `gcm` package (`~/workspace/gcm`), **not** the harness.
Training dynamics + method signals go to WandB **entity `uva-vast`, project `gcm`** (run `recipe-<model>-mix-<N>gpu`).

- **One experiment across all GPUs** (data-parallel, **not** one-bench-per-GPU): one process per GPU, per-GPU batch 1, adapter grads all-reduced each optimizer step ⇒ `effective_batch = world_size × grad_accum` (4×2 = **8**).
- **Loss = `answer_ce + distill_weight·distill_kl`** (each part logged separately, see the metric glossary): `answer_ce` = gold-answer CE from compressed `M` (read-LoRA on); `distill_kl` = KL(full-ctx teacher ‖ compressed student) over the gold answer.
- **Default recipe (env knobs of `experiments/run_recipe.py`):** `K=16`, `depth=half`, `proj_mult=2`, `read_lora=32`, `epochs=3`, `lr=3e-4` (AdamW, cosine+warmup), `distill_weight=0.5`, `grad_accum=2`, `n_train=96/bench`, `regime=mix` over `{bfcl_live_multiple, toolace, squad_v2, narrativeqa}`.
- **Launch:** `source /mnt/persist/.gcm_env && source /mnt/persist/.wandb_env; /mnt/persist/venv/bin/python -m torch.distributed.run --nproc_per_node=4 experiments/run_recipe.py /mnt/persist/checkpoints/Qwen3.5-9B`.
- **Metric glossary (every logged item explained):** [`gcm/docs/metrics.md`](../../../../../../../../gcm/docs/metrics.md) — groups `loss/ opt/ perf/ mem/ grad/ gate/ val/ eval/`. The headline method signal is **`gate/auroc_margin`** (can a label-free signal tell when to trust `M`).
- **Environment reproduction:** one-click `gcm/scripts/deploy.sh` (pins torch 2.11+cu128 / transformers 5.10 from `deploy/requirements-lock.txt`); see [`gcm/docs/ENVIRONMENT.md`](../../../../../../../../gcm/docs/ENVIRONMENT.md).
