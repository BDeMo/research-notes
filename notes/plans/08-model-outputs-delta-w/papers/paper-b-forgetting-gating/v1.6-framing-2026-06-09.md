# v1.6 framing: robust context compression for agentic tool-use (detect-and-fallback)

> Supersedes the v1.5 framing. v1.6 grounds the work in agentic deployment: the contribution is a
> harness-level circuit breaker that reads the model's INTERMEDIATE states to catch a broken context
> compression EARLY and fall back to full context. House rule: no em-dashes. Date: 2026-06-09.

## 1. Overall setting (why this matters)
Agentic AI improves UX, and its core is tool calling. Tool calling must be **fault tolerant**, or the
agent becomes unusable on many tasks, especially **long-context** tasks. But long context itself brings
problems:
- **complexity / cost**: attention and latency grow with context length;
- **loss of focus** (lost-in-the-middle, distraction by irrelevant spans);
- **context-window bottleneck**: the budget is finite.
So we **compress** the context (into latents) to relieve all three.

## 2. Significance (why compress, and the simple safety net)
- Why compress: efficiency + focus + fits the budget.
- The risk: compression is **lossy**, so it can drop critical information and make the agent fail,
  often silently. This is the do-no-harm problem.
- The simple, robust fix: **when the compressed context is (about to be) broken, do not compress, use
  the original full context.** Full context is the upper bound, so this fallback is guaranteed safe.
  The whole game becomes deciding *when* to invoke it.

## 3. The core question
**How do we judge, ideally EARLY, that a context compression has gone wrong**, so the harness can
interrupt and fall back to full context?

## 4. Why existing methods do not answer it (brief review)
- **(1) Input-time check is impossible in general.** No method can, *before* inference, tell whether
  the compressed latents lost critical information, **especially out of distribution**.
- **(2) Output / prompt-coupled checks are expensive and still blind.** Methods that judge by running the
  model and selecting (best-of-N: sample N completions, pick by a metric) pay for a long intermediate
  reasoning trace plus many pre-output samples, and **still have no oracle** to compare against, so they
  cannot actually verify the compression was right or which output to trust.
- Gap: neither *pre-input* nor *post-(many-)inference* reliably tells you whether compression is robust.

## 5. Our move (high-level motivation)
Since neither before-input nor after-many-full-inferences can settle which path is effective and
compression-robust, we make a **preliminary judgment from the model's intermediate states during a
single forward / decode pass.** By Occam's razor and harness-deployment practice, the stripped-down
core need is: **discriminate whether the model is compressing effectively.** If a compression failure is
detected early in inference, the **harness interrupts and uses full context as the robustness backstop.**
This is a deployment mechanism (a circuit breaker), not a new model.

## 6. Two mutually-exclusive scenarios
### Scenario (1): the knowledge location is KNOWN -> targeted verification
If we know which parameters hold which knowledge, we verify by watching **those modules' behavior** on a
given (context + prompt). Instantiations:
- **LoRA branch on a frozen base** (e.g. a branch on the MLP): new-dataset knowledge lives in the LoRA
  branch, so the branch's activation/behavior tells us whether to trust compression for this input.
- **MoE**: incrementally train an expert for the new knowledge and adjust the router; the router/expert
  behavior is the signal.
- **Experiment (1):** study the patterns of compression-effectiveness under mainstream LoRA / MoE
  methods, and extract the early interrupt signal from the branch/expert behavior.

### Scenario (2): the knowledge location is UNKNOWN -> relevance judgment
We cannot tell which retained span is "critical", so we need a **prompt-context relevance judgment**. A
**lightweight representation-level compressor** (simply trained) can largely provide it.
- **Recursion to (1):** the compressor is itself trained on some data, so we can apply Scenario (1) to
  judge whether the compressor actually acquired that knowledge, and extract the interrupt signal from
  the compressor's own modules. This keeps compression-and-use robust.
- **Experiment (2):** under a frozen base, train a lightweight compressor; build a **compressor library**
  over multiple training sets; study the compression-signal patterns and run comparison experiments.

## 8. The unifying mechanism (one sentence)
Both scenarios produce an **early interrupt signal read from the model's intermediate states**; the
harness uses it to route **keep the compressed context (cheap) vs interrupt and re-read full context
(robust)**. That is the do-no-harm gate, reframed as a circuit breaker for agentic tool-use, and the
do-no-harm floor (Section 2) is what makes the fallback safe.

## 9. Honest boundary (carried from v1.5)
Reading the interrupt signal per input is **imperfect** (per-input harm is hard to predict; rich signals
do not beat cheap base-uncertainty; MoE source-selection is at chance). So the design relies on the
**conservative fallback + the guaranteed floor**, not on a perfect detector. This honesty is the point:
the value is a safe, cheap, general circuit breaker, not an oracle.
