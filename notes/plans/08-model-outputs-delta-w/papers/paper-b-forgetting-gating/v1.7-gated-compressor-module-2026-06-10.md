## 0. What changes vs v1.6 (one paragraph)
v1.6 treated gating as a compressor-agnostic signal layer: read base uncertainty / divergence off any
compressor (Gist, Cartridge, ours) and ask "does the gate generalize across compressors". v1.7 makes the
gate a **structural part of our compressor**: a single module produces the compressed memory $M$ and a
calibrated reliability score $g$ from its **own internals** (no extra base forward to compute the signal),
and uses $g$ to decide compressed-vs-fallback at prefill. The two baselines stay exactly as their papers
define them (always-on, no gate). The headline object is therefore one **gated compressor module**, and
the main result is a clean compressor-level comparison. This tightens novelty to the module (not to a
probe) and is apples-to-apples for a reviewer.

## 1. The module: inputs and outputs
Input: context chunks $C=\{c_1,\dots,c_N\}$ (encoded by the **frozen** base into hidden states $h_t$) and
the query $q$. The module emits a pair, with no extra base forward for the signal:
$$ \big(M,\; g\big)=\mathrm{GCM}(C,q),\qquad M\in\mathbb{R}^{K\times d}\ \text{(compressed memory)},\qquad g\in[0,1]\ \text{(reliability)}. $$
$g$ is a **prefill-time** scalar (available right after compression, before decoding), so it can drive a
compute-saving fallback decision.

## 2. Compressor (kept from OURS, unchanged)
Frozen base encodes each chunk; an additive Perceiver-IO recurrence folds the chunks into $K$ latent slots:
$$ m' = m + \mathrm{MHA}(\mathrm{LN}(m),\mathrm{LN}(h_t),\mathrm{LN}(h_t)),\quad \Phi=m'+\mathrm{FFN}(\mathrm{LN}(m')),\quad m_t=m_{t-1}+\alpha_u\,\mathrm{LN}(\Phi-m_{t-1}),\ m_0=M_{\text{init}}. $$
$M=m_N$ (optionally one apply-time cross-attention readout over the full chunk pool). This already reads the
**base's hidden states** (it is not a bag-of-embeddings pooler), which is what makes the compressed memory
informative and the internal gate features meaningful.

## 3. The fused gate (new): three structural feature families -> one calibrated head
The gate reads only the module's **own internals** (plus the query), so it adds no base forward. Three
families, concatenated into one small MLP:

**(a) Reconstruction coverage $c_{\text{rec}}$ (did I keep the info?).** A tied lightweight decoder
reconstructs the context from $M$ (target = per-chunk pooled hidden); we use the residual statistics
$$ c_{\text{rec}}=\big[\,-\overline{\lVert \mathrm{Dec}(M)-\mathrm{pool}(h)\rVert},\ \ -\max_t\lVert \mathrm{Dec}(M)_t-\mathrm{pool}(h_t)\rVert\,\big]. $$
The mean term = global coverage; the max term = worst-covered chunk (the part most likely to break recall).
This is **label-free** and falls straight out of the autoencoding objective the compressor already trains on
(`reconstruction_head` exists today).

**(b) Query coverage $c_{q}$ (is the kept info relevant to THIS query?).** With $\bar q=\mathrm{pool}(E_q)$
and attention weights $a_k=\mathrm{softmax}_k(\bar q\!\cdot\!M_k/\sqrt d)$:
$$ c_{q}=\big[\,\max_k \cos(\bar q,M_k),\ \ \cos(\bar q,\mathrm{pool}(M)),\ \ H(a),\ \ \max_k a_k\,\big]. $$
Diffuse attention / low max-cosine = the query cannot find what it needs in $M$ (the foreign-context /
irrelevant case the agent framing needs).

**(c) Memory geometry $\phi_{\text{geo}}$ (cross-model-general signals, validated in the 7-model study).**
$$ \phi_{\text{geo}}=\big[\,\overline{\lVert s\rVert},\ \max\lVert s\rVert,\ \mathrm{drift}=\lVert m_N-m_0\rVert,\ \lVert\Delta m_{\text{last}}\rVert\,\big]. $$

**Gate:** $\,g=\sigma\big(\mathrm{MLP}([\,c_{\text{rec}},\,c_{q},\,\phi_{\text{geo}}\,])\big)$. One head, calibrated (§4).
(Optionally a single cheap base-uncertainty scalar `margin_0` can be appended as a feature for Track B; it is
one extra no-context draft, not a full re-read. Default keeps the gate purely internal.)

## 4. Training (only the module trains; base stays frozen)
Per item, with the memory injected via the chosen combine:
$$ \mathcal{L}=\underbrace{\mathrm{CE}(y^\star\mid M,q)}_{\text{task}} + \lambda_{\text{kl}}\underbrace{\mathrm{KL}\big(\text{base}_{\text{no-ctx}}\,\Vert\,\text{base}_{M}\big)}_{\text{do-no-harm reg}} + \lambda_{\text{rec}}\underbrace{\lVert \mathrm{Dec}(M)-\mathrm{pool}(h)\rVert^2}_{\text{reconstruction (feeds }c_{\text{rec}}\text{)}} + \lambda_{\text{gate}}\underbrace{\mathrm{BCE}(g,\,y_{\text{dnh}})}_{\text{gate calibration}} + \lambda_{\text{div}}\,\mathcal{L}_{\text{div}}. $$
The **gate-calibration label** is the do-no-harm target for the deployed fallback:
- Track A (vs no-context): $y_{\text{dnh}}=\mathbb{1}[\,n_w\ge n_0\,]$ (memory at least as good as the bare base).
- Track B (vs full-context): $y_{\text{dnh}}=\mathbb{1}[\,n_w\ge n_{\text{full}}-\varepsilon\,]$ (compression suffices).

$n_0,n_w,n_{\text{full}}$ = native score with no-ctx / memory / full-context (known at train time). We can
either train a single $g$ for one chosen track or a 2-logit head $(g_A,g_B)$. The gate thus **learns to
predict its own reliability** from the structural features, which is the "clever structural fusion": the
decision signal is produced by the compressor, not by probing the base after the fact.

## 5. Deploy: detect-and-fall-back (the headline mechanism)
At inference compute $M$ and $g$ at prefill, then route per input:
$$ \text{output}(x)=\begin{cases}\text{base}_{M}(x) & g\ge\tau\ \ (\text{use the cheap compressed memory})\\[2pt] \text{fallback}(x) & g<\tau\ \ (\text{robust path})\end{cases} $$
- **Track A (do-no-harm / forgetting):** fallback = bare base. Property: never worse than the base when the
  gate is correct; with $g\to0$ the injection vanishes and the output is the **exact** frozen base (do-no-harm
  by construction).
- **Track B (compression / acceleration):** fallback = re-read full context. Metric = accuracy at token cost
  ("recover X% of full-context accuracy at Y% of tokens"); compressed budget $\approx K+|q|$ tokens.

## 6. Evaluation objective: the compress-decision confusion matrix (precision / recall / F1)
With a fall-back available, the system accuracy is **floored by the fall-back** (full context). So the thing
we optimize is NOT raw accuracy; it is the **quality of the compress / fall-back decision**: compress as often
as is safe. Treat the gate as a binary classifier, positive action = "compress" (use the cheap memory $M$).
The latent truth is whether compression is acceptable for this input,
$y=\mathbb{1}[\,n_w\ge n_{\text{full}}-\varepsilon\,]$ (Track B; Track A uses $n_w\ge n_0$). Four cells:

| | compression acceptable $y{=}1$ | compression fails $y{=}0$ |
|---|---|---|
| **decide compress** | TP: cheap and correct (ideal) | FP: cheap but wrong = the harm (do-no-harm violation) |
| **decide fall back** | FN: paid full cost when cheap would do (compute waste, no accuracy harm) | TN: correctly fell back, harm avoided |

Mapping to the three goals:
- **"compress as much as possible"** = **recall** $=\tfrac{TP}{TP+FN}$ (of the genuinely-compressible inputs, capture as many as possible).
- **"compress accurately"** = **precision** $=\tfrac{TP}{TP+FP}$ (when we compress, rarely wrong = do-no-harm).
- **"improve when not compressing"** = the **TN** payoff: on inputs where compression would fail, falling back recovers full-context accuracy (the FP we avoided).
- **judgment metric** = **F1**, which is exactly the classifier the GCM gate trains on via the §4 calibration BCE.

**The errors are asymmetric:** FP costs **accuracy** (harm), FN costs only **compute**. So the operating point is
usually **max recall subject to precision $\ge p^\*$** (do-no-harm constraint, e.g. $p^\*{=}0.95$) or $F_\beta$
with $\beta<1$, not plain F1. The full picture is the **cost-coverage / risk-coverage frontier** swept by the
threshold $\tau$ (lower $\tau$ = more coverage and cheaper, but lower precision = more harm); F1 summarizes one
operating point, AURC summarizes the curve.

**Anchors per bench:** base rate $P(y{=}1)$ = "compressibility"; always-compress (recall 1, precision = base
rate, max harm); always-fall-back (no compression, full cost); **oracle** (P=R=1 = ceiling + the compute it
saves). The gate's value = how close to oracle.

### Step-by-step program (improve each independently, report F1 each time)
1. **Define + measure (no model change).** Lock $\varepsilon$ + Track A/B labels; build a reporter over the
   already-logged $n_0/n_w/n_{\text{full}}$ + gate features that prints the confusion matrix, precision /
   recall / F1, cost-coverage / risk-coverage, and oracle + trivial-gate baselines, per bench. Shows the
   current gate's F1 and the oracle ceiling **before** any redesign. **Start here** (uses existing logs).
2. **Precision first (do-no-harm).** Threshold for precision $\ge p^\*$; report recall@$p^\*$. FP is the enemy.
3. **Recall at fixed precision (better gate).** Add the fused structural features (recon coverage + query
   coverage + geometry) + calibrated head; push the P-R curve out (higher recall at $p^\*$, higher F1, lower
   AURC). Ablation that justifies full fusion: geometry-only < +query-coverage < +recon-coverage.
4. **Raise the base rate (better compressor).** Better compression (distillation / capacity) so $P(y{=}1)$
   rises: more compressible inputs to capture, frontier shifts out. Independent of the gate.

## 7. Compressor-level comparison (drop "compressor as a setting")
Same frozen base, same $K$, same training data, same benches. The method and baselines are simply **rows**,
not a setting axis:

| row | what it is | gate? |
|---|---|---|
| `no-ctx` | bare base, query only | floor |
| `Cartridge` (vanilla, Eyuboglu+ 2025) | per-corpus cartridge by self-study/distillation, **always used** | none |
| `Gist` (vanilla, Mu+ 2023) | gist tokens + gist mask, **always used** | none |
| **`OURS` (GCM)** | gated compressor: compress + detect-and-fall-back | **intrinsic** |
| `full-ctx` | full context in the prompt | ceiling |

**Claims the table makes.**
1. At a compressed token budget, `OURS` matches the best vanilla compressor on accuracy **and** never falls
   below its fallback when the gate fires (do-no-harm), while vanilla `Cartridge` / `Gist` show negative
   transfer (compression hurts) with **no recourse**.
2. The gate is **what makes the compressor safe**: the key ablation is `OURS` with the gate removed, which
   degrades to a vanilla compressor and reproduces the baselines' negative transfer.
3. The in-domain / cross-domain split (in-task, cross-task-in-domain, cross-task-cross-domain) shows the gate
   closing as the eval moves away from the training corpus, so the average cross-domain score is preserved.

**We do not gate the baselines.** They are the original always-on compressors, which is the honest
comparison. The "gate generalizes across compressors" experiment is **dropped** (it implied gating was a
separable add-on; v1.7 says it is part of our module).

## 8. What this drops / changes from v1.5-v1.6
- DROP: compressor-agnostic gate probes on Gist/Cartridge (`--gate/--moe/--curve/--bon` plumbing on the
  baseline path), and the gate-generality framing.
- KEEP, RELOCATE: the validated gate signals (`delta_last`, slot geometry, base-uncertainty) become the
  **geometry family** inside the fused gate; query-relevance becomes the **query-coverage family**.
- ADD: reconstruction-coverage family + the joint **gate-calibration** training objective, so $g$ is learned
  end to end as a reliability predictor rather than fit offline by a separate logistic CV.
- The two-track design (A vs no-ctx, B vs full) is unchanged in spirit; it is now realized by the single
  module's $g$ (or a 2-logit head).

## 9. Honest positioning / novelty
The compressor mechanisms are not ours (cite Gist, Cartridges, ICAE). The vanilla-vs-vanilla rows are the
established compressors. Our contribution is the **gated compressor module**: a compressor that **predicts
its own reliability from its internal structure and falls back when compression is unsafe**, giving a
do-no-harm floor by construction on a frozen base. This is narrower and more defensible than "a gate that
works on any compressor"; the reviewer sees one module and a clean comparison.

## 10. Implementation plan (only after you approve the architecture)
1. `wrapper.py`: extend `_GateHead` to consume the full fused feature set (add reconstruction-coverage from
   the recon head residual + query-coverage attention stats; keep geometry). Make the recon head on by
   default for GCM and expose its residual to the gate. Expose $g$ at apply time + a fallback-decision hook.
2. OURS trainer: add $\mathcal{L}_{\text{gate}}=\mathrm{BCE}(g,y_{\text{dnh}})$ and ensure $\mathcal{L}_{\text{rec}}$ is on;
   compute $y_{\text{dnh}}$ per item from $n_w$ vs the track fallback.
3. Comparison harness: one script that builds the §7 table per bench, with `OURS` applying detect-and-fall-back
   (tracks A/B) and the baselines vanilla. Reuse `mem_baselines.py` for vanilla Cartridge/Gist.
4. Strip the gate/signal plumbing off the baseline path (keep only `native_0 / native_w / native_full`).

## 11. Open questions / risks
- **Reconstruction target granularity:** coarse pooled-context vs per-chunk reconstruction (per-chunk gives a
  sharper worst-covered-chunk feature but costs a small decoder). To finalize at implementation.
- **One $g$ vs 2-logit $(g_A,g_B)$:** a shared trunk with two heads is cheap and lets one module serve both
  tracks; default to 2-logit.
- **Calibration vs accuracy trade-off:** $\lambda_{\text{gate}}$ must not dominate the task loss; tune so the
  compressor stays strong and $g$ stays calibrated (report reliability diagrams + risk/cost-coverage).
- **Gate degeneracy:** still must check $g$ does not collapse to a constant (always-fallback would trivially
  do no harm). Report use-rate / help-rate / precision-recall as before.
