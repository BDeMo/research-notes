## TL;DR

A 218 M-trainable wrapper on a **frozen** Qwen3-8B base, trained on **one** synthetic compression task (`categorical_niah`), characterises a **three-regime transfer law** on 8 fully public benchmarks (zero-shot transfer):

- **Regime A — lossy-compression wins** (QuALITY, 4-way MC): both wrappers beat all four no-training baselines by +5 to +12 pp; OURS ≈ Gist within seed noise but emits 32 vs 48 soft tokens (1/3 fewer).
- **Regime B — at-chance** (MuSR-mm, multi-step reasoning over closed-MC narratives): both wrappers ≈ no_context; full_context wins. No transferable reasoning signal.
- **Regime C — wrapper collapses to zero** (RULER-NIAH, verbatim retrieval): OURS = GIST = **0.000 ± 0.000** (4 seeds, both exact zero); full_context = 0.995. Lossy compression literally destroys the needle string. Cleanest possible negative result.

This replaces both the earlier v1.2 "OURS > Gist +12 pp" headline (seed-42 artifact) and the v1.3 "OURS ≈ Gist" headline (single-bench). Honest, tight, falsifiable.

## 2. Locked architecture

Memory state $m_t \in \mathbb{R}^{K \times d}$, K = 32 soft tokens, d = 4096 (Qwen3-8B hidden) / 5120 (Qwen3-14B).

**Write side (recurrence):**

$$\Delta m_t = \mathrm{LayerNorm}(\tilde m_t - m_{t-1}),\qquad m_t = m_{t-1} + \alpha_\text{upd}\cdot \Delta m_t$$

- $\tilde m_t = f_\theta(m_{t-1}, c_t)$ via 1 Perceiver-IO-style cross-attention + FFN block (canonical: L=1, h=4).
- $\alpha_\text{upd}$ is a **learnable scalar**, init from CLI.
- $\Delta m_t$ exposed on `MemoryState.extra["last_delta"]` for geometry probes.

**Read side (combine).** 7 modes total. Default since Phase F = **`xattn`** (`xattn_residual` available, more expressive but no clear lift over `xattn` in final recipe).

| mode | $T'$ | mechanism |
|---|---|---|
| `prefix` (control) | $K+T$ | $[m_T, x]$ |
| `residual` | $T$ | per-token $h \mathrel{+}= \alpha_\text{app}\cdot \mathrm{xattn}(h, m_T)$ |
| `interleave` | $K+T$ | insert $m_T$ at $\lfloor T\cdot p\rfloor$ |
| `hybrid` | $K+T$ | residual *then* prefix |
| **`xattn` ★** | $K+T$ | refine $m_T$ via xattn over full raw chunk pool, then prefix |
| `xattn_residual` (Phase M) | $T$ | refined $m_T'$ via xattn then per-token residual |
| `gated` | — | placeholder; `NotImplementedError` |

**Direct answer head** (Phase Q, 2026-06-02): `Linear(d, n_classes)` on `pool(m_T)`. Bypasses the frozen-base CE bottleneck for categorical tasks. Loss term $\lambda_\text{ah}\cdot \mathrm{CE}(\mathrm{head}(\mathrm{pool}(m_T)), \text{gold class})$. Risk: makes $m_T$ class-label-like rather than content-like; kept as auxiliary, not primary.

**Trainable parameter count: 218 M (2.67 % of base).** **Not** a small adapter. Positioned in the soft-prompt / Perceiver-IO family where ~100–300 M trainable is standard. Word "lightweight" dropped from the abstract.

## 3. Training (final v1 recipe collapsed to SFT-only)

Designed as 3 stages; OPD and RL added no measurable lift on Phase Y cells → **shipped as SFT-only**.

| stage | trainer | primary loss | status in v1 final |
|---|---|---|---|
| SFT | `BPTTTrainer` | $L_\mathrm{ce} + \lambda_\mathrm{kl}L_\mathrm{kl} + \lambda_\mathrm{div}\cos(\Delta m_t, \Delta m_{t-1}) + [\lambda_\mathrm{recon}\,\mathrm{MSE}] + [\lambda_\mathrm{ah}\,L_\mathrm{ah}]$ | **kept** |
| OPD (Phase E) | `OnPolicyDistillTrainer` | masked CE between teacher distribution and student distribution on student's own rollouts | dropped (no lift; OPD-alone collapses without SFT warmup) |
| RL | `RLTrainer` | REINFORCE on $r \in \{\text{contains}, \text{exact}\}$, leave-one-out baseline + KL anchor | dropped (no lift) |

**Canonical recipe** (Phase Y, in `k8s/sweep/build_sweep.py::BEST_RECIPE + BEST_ARCH`):

- Base: Qwen3-8B frozen, bf16
- K = 32, h = 4, L = 1 (single update block)
- combine = `xattn`
- N items = 500 train, 50 held-out
- Steps = 1800, AdamW
- $\lambda_\mathrm{div} = 0.1$ (goldilocks; 0.5 underfits, 0 lets drift explode)
- $\lambda_\mathrm{ah} \approx 0.5$ for categorical tasks
- 4 seeds: {42, 7, 11, 13}

Auxiliary losses added in Phase M / Q:

- **Reconstruction loss** (Phase M): $\|\mathrm{head}_R(\bar m_T) - \bar h_\mathrm{chunks}\|^2$ forces $m_T$ to carry summarized content even when answer-CE alone fails.
- **Consecutive-Δm penalty** (Phase A): pushes $\cos(\Delta m_t, \Delta m_{t-1})\to 0$. Stops the recurrence from collapsing to a constant direction.

## 5. THE 3-REGIME LAW — headline result (Phase Y, 4 seeds)

| benchmark | metric | OURS μ±σ (n=4) | GIST μ±σ (n=4) | best baseline | regime |
|---|---|---|---|---|---|
| **QuALITY-val** | `accuracy_letter` | **0.193 ± 0.032** | 0.180 ± 0.044 | no_context 0.141 | **A** — wrapper wins |
| **MuSR-mm** | `accuracy_letter` | 0.493 ± 0.008 | 0.501 ± 0.013 | full_context 0.551 | **B** — at-chance |
| **RULER-NIAH** | `exact_value_match` | **0.000 ± 0.000** | 0.000 ± 0.000 | full_context 0.995 | **C** — collapse |

Per-seed RULER-NIAH (the strictest test): OURS{42,7,11,13} = GIST{42,7,11,13} = **{0,0,0,0}** exactly. Zero variance, zero signal. Lossy compression destroys the needle string.

Source: `runs/sweep-phaseY-20260603-195211UTC/` + `runs/sweep-phaseY-rulerassist-20260603-205145UTC/`.

**Falsifiable v1 paper claim** (replaces all earlier headlines):

> *"A 218 M-trainable wrapper on a frozen 8 B base, trained on one synthetic compression task, characterises a three-regime transfer law: on benchmarks where the answer can be reconstructed from a lossy gist of the context (QuALITY), the wrapper matches Gist (SOTA soft-prompt baseline) and beats all no-training baselines by +5 to +12 pp using 32 vs 48 soft tokens; on benchmarks requiring verbatim retrieval (RULER-NIAH), it collapses to zero while full_context retains 0.995; on reasoning tasks (MuSR-mm), neither wrapper exceeds chance."*

## 7. Engineering findings worth carrying forward

- **Bit-capacity wall** is a real, reproducible phenomenon. Phase I/K/P curves at $d \in \{1..12\}$ show a sharp transition; scale-invariant across Qwen3-8B and Qwen3-14B (Phase O / P).
- **Indirect-supervision pathology**: at small N, wrapper collapses to per-item memorization (train_em ≈ 0.98, eval_em ≈ 0.24). Data scale (Phase L `data_500`) is what broke out — *not* optimizer steps or stage stacking.
- **OPD-alone collapses** without SFT warmup. Stage order in the 3-stage pipeline matters; OPD on top of SFT was the only non-degenerate non-pure-SFT recipe found, but ultimately added no measurable lift over SFT-only at Phase Y.
- **Answer head** is necessary for categorical tasks to fight the frozen-base CE bottleneck, but at $\lambda_\text{ah} \to 1$ the wrapper learns to predict a single high-frequency class (Phase Q early-smoke: identical em=contains=f1=0.540 caught this).
- **$\lambda_\mathrm{div}$ goldilocks at 0.1** — 0.5 underfits, 0 lets drift explode (e.g., ksweep_K128_d5 drift = 44 288).
- **`xattn` combine mode** wins on `coding_niah` (where the answer is a specific 8-char string) and ties on categorical tasks. It is the only mode where the wrapper has a second pass over the *raw* chunk pool right before the base sees the prefix.

## 8. v2 design (next paper)

**v2 = v1 + 3 optional protocol methods.** Zero breaking change to v1.

```python
def update_from_generation(m, gen_step) -> MemoryState
def serialize(m) -> bytes
def load(blob) -> MemoryState
```

Key shift: **suffix memory after autoregressive generation** (vs v1's prefix-only encode-time memory). Cross-session: write in session 1, query in session 2.

- **Trigger choice**: end_of_turn (default), sentence, every_n, or model-emitted `<memwrite>` markers.
- **Datasets target**: LongMemEval, LOCOMO (real conversational traces).
- **Head-to-head baseline**: **TokMem** (reimplement on top of this codebase as v2 main baseline).
- **Open question**: does the bit-capacity wall hold at *write-time* (after generation) the same way as at *encode-time*?
- **Timeline**: v1 numbers frozen 2026-06-15; v2 first draft ~2026-08; v2 PRs blocked until then.

Engineering scope: `GenStep` dataclass + `SuffixSessionTrainer` (1d) → `update_from_generation` reusing existing update block (2d) → `serialize`/`load` bf16 dump ~256 KiB per session (0.5d) → `multi_session` synthetic dataset (1d) → Phase M smoke test (2d) → first v2 capacity curve (3d). Then real-text benchmarks (LongMemEval / LOCOMO, 1 week each) + TokMem reimpl (1 week).

## 10. Operational lessons (transferable to other plans / repos)

These already burned us; documenting so future plans don't repeat.

- **Atomic-swap-sync postmortem** (2026-06-02): `rm -rf REMOTE; mv REMOTE.new REMOTE` during a live training job orphans the worker's cwd inode → every subsequent relative-path `mkdir runs/...` fails with `FileNotFoundError`. **Fix**: coerce all output paths to absolute via `Path(...).resolve()` at the top of `main()`; refuse-to-sync guard in `k8s/sync-to-sam-dev.sh` that aborts if any `python.*train_smoke.py` is running on the pod.
- **Mid-sweep MKL crash**: code sync mid-sweep corrupts the CUDA context when the GPU worker forks a new Python process → `Intel MKL FATAL ERROR: Cannot load libtorch_cpu.so`. Single-GPU cells that started fresh after the sync were unaffected. **Mitigation**: same sync-guard above.
- **Waiter-chain bugs** (Phase L→M→NOP→Q sequence, ~13 h lost): (i) waiter passed `--jobs` to a script expecting `--config`; (ii) downstream waiter matched a `*.orchestrator.log` *file* as the *directory* and polled it forever; (iii) waiters used bare `python3` instead of conda env. **Hardening**: `find -type d`, conda python, `--python-bin`, `mkdir -p`, absolute `OUT_DIR`.
- **Phase end-crash that left data on disk**: `KeyError: 'drift_curve'` for gist wrappers crashed at the end of training but the per-strategy `summary.json` files had already landed. **Lesson**: every aggregator should *also* be able to recover from per-file artifacts on disk, not just trust the orchestrator's `status.jsonl`.
- **Sweep manifest overwrite on relaunch** (unresolved): when a fresh orchestrator launches into an existing sweep directory it overwrites `runs_manifest.json`. **Fix needed**: merge instead of overwrite.
