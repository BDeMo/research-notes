\begin{frame}{Background: What RCA Means}
  \begin{itemize}
    \item \textbf{RCA}: root-cause analysis identifies the failing service,
      fault type, supporting evidence, confidence, severity, and recommended fix.
    \item RCA is useful for incident response because it turns noisy operational
      evidence into an actionable diagnosis and recovery recommendation.
    \item An RCA case can include incident symptoms, service topology, logs,
      traces, fault reports, historical cases, and human-written summaries.
    \item The model must do more than answer a question: it must connect evidence
      to cause, keep output structure, and explain what should be fixed.
  \end{itemize}
\end{frame}

\begin{frame}{Significance: Why an RCA Foundation Model}
  \begin{itemize}
    \item General LLMs have strong language ability, but they are not specialized
      for structured RCA outputs or service-level diagnosis.
    \item An RCA foundation model should provide a reusable base capability for:
      \begin{itemize}
        \item root-cause identification;
        \item evidence extraction and grounding;
        \item fault / severity / confidence classification;
        \item fix recommendation generation.
      \end{itemize}
    \item The goal is not only a demo UI; the goal is to train a model with
      reusable RCA capability.
  \end{itemize}
\end{frame}

\begin{frame}{Main Goal: Train an RCA Foundation Model}
  \begin{itemize}
    \item Start from existing open-source pretrained models.
    \item Train them on RCA-style data so they learn domain behavior that the
      original pretrained model does not reliably have.
    \item Use public RCA datasets as the core training/evaluation source:
      \begin{itemize}
        \item Nezha;
        \item OpenRCA-500;
        \item RCAEval;
        \item lincyaw/rca.
      \end{itemize}
    \item Use a small internal RCA dataset only as an internal qualitative probe,
      not as a public release artifact.
  \end{itemize}
\end{frame}

\begin{frame}{Deliverables: What We Have Built}
  \begin{itemize}
    \item Trained RCA-capable model checkpoints from open-source base models.
    \item Built an RCA SFT matrix over:
      \begin{itemize}
        \item two base families: Qwen2.5-7B-Instruct and Qwen3-8B;
        \item curriculum vs. joint-mix training;
        \item four public RCA datasets;
        \item five general benchmarks for capability-retention checks.
      \end{itemize}
    \item Built `rca-chat`, a Gradio probe to load checkpoints and inspect
      qualitative behavior on RCA cases.
    \item Packaged matrix reports, provenance notes, and project brief slides.
  \end{itemize}
\end{frame}

\begin{frame}{Current Capability: What Improved}
  \begin{itemize}
    \item The trained model can learn RCA-specific behavior that the base model
      does not consistently produce.
    \item Observed improvements include:
      \begin{itemize}
        \item more structured RCA JSON outputs;
        \item better service-level root-cause identification;
        \item stronger fault / severity fields on public RCA benchmarks;
        \item more useful fix recommendation text.
      \end{itemize}
    \item The chat probe makes these changes visible case by case, while the
      matrix shows the quantitative trend.
  \end{itemize}
\end{frame}

\begin{frame}{Future Problem: Catastrophic Forgetting}
  \begin{itemize}
    \item The largest research issue is \textbf{catastrophic forgetting}.
    \item Definition: after training a pretrained model on new RCA data, the model
      may lose capability learned from earlier RCA datasets or from pretraining.
    \item This happens because our RCA data is much smaller than the original
      pretraining corpus; we cannot fully protect the base model by simply
      mixing in all original data.
    \item In our setting, forgetting can show up as:
      \begin{itemize}
        \item worse performance on earlier RCA datasets after later training;
        \item valid JSON with missing evidence fields;
        \item format collapse after LoRA updates;
        \item drops on general benchmarks such as reasoning or instruction following.
      \end{itemize}
  \end{itemize}
\end{frame}

\begin{frame}{Future Work: How We Are Trying to Solve It}
  \begin{itemize}
    \item We already compared multiple training strategies:
      \begin{itemize}
        \item sequential curriculum training;
        \item joint-mix training across RCA datasets;
        \item anti-forget v2 with data balancing and a small general anchor.
      \end{itemize}
    \item Early conclusion: direct SFT is useful for initializing RCA capability,
      but it is not stable enough as the final answer.
    \item Next research focus: keep the base model more stable and move
      adaptation into memory / wrapper mechanisms.
    \item Next demo: compare full context, summary, retrieval, and
      compressed-memory RCA on the same public RCA case.
  \end{itemize}
\end{frame}
