\begin{frame}{This Week: Main Message}
  \begin{block}{High-level direction}
    Controlled model adaptation: improve RCA / long-context capability while
    preserving the frozen base model's general ability.
  \end{block}
  \begin{itemize}
    \item \textbf{Plan 08 v1}: learned soft-prompt memory is now framed as a
      lossy-compression characterization, not a universal-memory claim.
    \item \textbf{Plan 08 v1.5}: next step is a do-no-harm gate that opens the
      wrapper only when compression is likely safe.
    \item \textbf{Plan 08 v2}: next version is cross-session latent memory with
      read/write tokens under a frozen base.
    \item \textbf{Related direction}: Janus tests whether long-context sites
      and forgetting-sensitive sites are coupled.
  \end{itemize}
\end{frame}

\begin{frame}{Problem Statement: Why This Matters}
  \begin{itemize}
    \item RCA and debugging inputs are long: logs, traces, topology, prior
      observations, and code context often exceed cheap prompting.
    \item Direct SFT can teach RCA behavior, but risks weakening general
      instruction-following, reasoning, or retrieval behavior.
    \item The safer target is a \textbf{frozen-base adaptation module}: learn
      context compression / memory without overwriting the base model.
    \item This week sharpened the decision rule: use memory when it helps;
      close it when it would hurt.
  \end{itemize}
\end{frame}

\begin{frame}{Goal and Significance}
  \begin{columns}[T]
    \begin{column}{0.48\textwidth}
      \begin{block}{Goal}
        Train a small wrapper around a frozen base model so long context can be
        compressed into a reusable memory signal without updating base weights.
      \end{block}
      \begin{itemize}
        \item Input: long evidence context + query.
        \item Output: answer from the frozen base, optionally helped by wrapper memory.
      \end{itemize}
    \end{column}
    \begin{column}{0.48\textwidth}
      \begin{block}{Why it matters}
        RCA needs adaptation to domain evidence, but direct SFT can damage
        general ability. The wrapper is a controlled adaptation module.
      \end{block}
      \begin{itemize}
        \item Learn useful compression when it is safe.
        \item Fall back when compression would lose details.
      \end{itemize}
    \end{column}
  \end{columns}
\end{frame}

\begin{frame}{Comparison: What We Are Really Testing}
  \small
  \begin{tabular}{p{0.24\textwidth}p{0.31\textwidth}p{0.34\textwidth}}
    \toprule
    Approach & Benefit & Limitation \\
    \midrule
    Full context & Preserves exact details & Expensive; not persistent \\
    Direct SFT & Learns task behavior & Can regress base capabilities \\
    Retrieval only & Good for verbatim facts & Does not learn compression / synthesis \\
    Always-on wrapper & Learns compressed memory & Can hurt unseen or exact-retrieval cases \\
    \textbf{Gated wrapper} & Uses memory only when safe & Needs reliable gate signals \\
    \bottomrule
  \end{tabular}

  \vspace{0.7em}
  \begin{block}{Positioning}
    We are not claiming ``wrapper replaces context.'' We are testing controlled
    adaptation: train a wrapper, measure where it helps, and gate it off when it
    would damage the frozen base path.
  \end{block}
\end{frame}

\begin{frame}{Core Evidence Chain}
  \begin{enumerate}
    \item \textbf{Wrapper can train and learn something.}
      It improves compressible / gist-style cases relative to no-context
      baselines.
    \item \textbf{But learned memory changes inference.}
      The wrapper is not neutral: on some tasks it shifts the base away from
      useful behavior.
    \item \textbf{Unseen or exact-detail tasks are unsafe for forced memory.}
      If the task needs verbatim evidence or reasoning transfer the wrapper did
      not learn, always-on memory can lose the answer.
    \item \textbf{Solution: gate between wrapper and base.}
      Open the wrapper when confidence is high; close it when divergence/drift
      says the wrapper is fighting the base.
  \end{enumerate}
\end{frame}
