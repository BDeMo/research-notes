\begin{frame}{Problem: CMG-RCA Is a Deployment-Scale Long-Context Task}
  \small
  \begin{itemize}
    \item A CMG-RCA case can contain logs, telemetry, traces, tech-support files,
      previous reports, and retrieved evidence.
    \item Full raw evidence can be near or above the practical serving window.
    \item Full-context inference can hit GPU memory limits; truncation can remove
      the evidence needed for RCA; no-context has little CMG-specific prior.
  \end{itemize}
  \begin{block}{Goal}
    Extract useful RCA knowledge from long evidence in a deployable way, then
    decide whether that extracted knowledge is reliable enough to use.
  \end{block}
\end{frame}

\begin{frame}{v1.8.0 Outcome: Extracted Knowledge Beats Truncation}
  \small
  \begin{table}\centering
    \begin{tabular}{lc}
      \toprule
      setting & CMG module-ID accuracy \\
      \midrule
      no context & 0.079 \\
      truncated full context & 0.105 \\
      \textbf{extracted knowledge} & \textbf{0.263} \\
      \bottomrule
    \end{tabular}
  \end{table}
  \begin{itemize}
    \item The absolute number is early-stage.
    \item The important result is the ordering:
      \texttt{extracted knowledge > truncated full context > no context}.
    \item This validates the deployment motivation: under a memory wall, the
      extracted-knowledge path is more useful than simply truncating raw evidence.
  \end{itemize}
\end{frame}

\begin{frame}{v1.8.0 Caveats}
  \small
  \begin{itemize}
    \item Internal CMG is used as the demo and product motivation, not as the main
      paper benchmark.
    \item Some early CMG numbers may be optimistic if trained before module-name

    \item External generalization is not solved yet.
    \item The current result is small-scale and mostly single-turn.
  \end{itemize}
  \begin{block}{Implication}
    CMG proves the deployment need. The paper should be validated on public

  \end{block}
\end{frame}

\begin{frame}{Public Long-Context Fact-Base}
  \small
  \begin{block}{Question}
    Which long-context coping method works, when, and why?
  \end{block}
  \begin{itemize}
    \item Baselines: window / truncation, KV eviction, prompt compression, RAG,
      full-context and no-context references.
    \item The goal is not to crown one winner, but to map failure modes across
      length, task type, and model architecture.
    \item Paper claims should be grounded in public benchmarks; CMG remains the
      internal motivating demo.
  \end{itemize}
\end{frame}

\begin{frame}{Key Long-Context Facts}
  \small
  \begin{itemize}
    \item \textbf{Attention-based KV eviction can collapse past 8k--16k.}
    \item \textbf{Attention-free or reconstruction-style eviction is more length-robust on retrieval.}
    \item \textbf{Ranking flips by task type:} retrieval, extractive QA, global reasoning,
      and literary MC behave differently.
    \item \textbf{Full context can hurt} when distractors dominate.
    \item \textbf{RAG is strong with lexical overlap,} but weak for abstractive or global tasks.
    \item \textbf{Linear-attention / GDN-style models do not expose a KV lever.}
  \end{itemize}
  \begin{block}{Conclusion}
    A fixed context-reduction method is not enough.
  \end{block}
\end{frame}

\begin{frame}{v2.0.0 Paper Direction}
  \small
  \begin{block}{High-level thesis}
    Long-context RCA and agentic reasoning need adaptive knowledge extraction,
    not one fixed compression or retrieval rule.
  \end{block}
  \begin{itemize}
    \item Full context can be too expensive or misleading.
    \item Truncation only works when the answer is local.
    \item RAG only works when lexical overlap is available.
    \item KV eviction is model-architecture dependent and can fail at long length.
  \end{itemize}
  \begin{block}{Method framing}
    Extract useful knowledge from long evidence; verify whether it is reliable;
    fall back when it is not.
  \end{block}
\end{frame}

\begin{frame}{v2.0.0 Ideas}
  \small
  \begin{enumerate}
    \item \textbf{Self-verified extracted memory:} train the memory to preserve evidence;
      use that preservation/reconstruction signal as the reliability check.
    \item \textbf{Reliability-gated fallback:} use extracted knowledge when safe;
      otherwise fall back to raw context, retrieval, or a higher-cost path.
    \item \textbf{Adaptive budget:} easy contexts get fewer memory tokens; dense or
      ambiguous contexts get more.
    \item \textbf{Public benchmark story:} use public long-context tasks for paper
      claims; keep CMG as the applied internal demo.
  \end{enumerate}
\end{frame}
