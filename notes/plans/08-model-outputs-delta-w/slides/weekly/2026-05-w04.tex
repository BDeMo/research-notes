\begin{frame}{Background}
  \begin{itemize}
    \item RCA and debugging tasks are becoming long-context problems: logs, traces, stack traces, test failures, code context, topology, and prior observations often exceed a practical prompt budget.
    \item Direct long-context prompting is expensive and brittle: high prefill cost, large KV cache, lost-in-the-middle behavior, and no persistent learning between chunks.
    \item Directly fine-tuning an 8B base on our small RCA/debug data is risky because we do not have the original pretraining mix needed to protect the base model's general capability.
    \item This project starts from a frozen 7B/8B open-source base and trains a separate wrapper memory module.
  \end{itemize}
\end{frame}

\begin{frame}{Significance}
  \begin{itemize}
    \item \textbf{Research value}: formulate long-context compression as next-state prediction over learned memory states:
      \[
        m_t = G_\phi(m_{t-1}, c_t), \qquad y_t = F_\theta(q_t, m_t)
      \]
    \item \textbf{Product value}: build an RCA foundation model for structured root-cause analysis, with lower context cost and better evidence retention.
    \item \textbf{Risk control}: train only the wrapper in v0, keeping the base model frozen to reduce catastrophic forgetting.
    \item \textbf{Community impact}: release a model, benchmark protocol, demo, and technical report rather than only internal experiment logs.
  \end{itemize}
\end{frame}

\begin{frame}{RCA-Demo Initialization: SFT Boundary Search}
  \begin{itemize}
    \item Goal: use \texttt{rca-demo} to find how far standard SFT can push a general model toward RCA behavior.
    \item Setup:
      \begin{itemize}
        \item Displayed base family: Qwen2.5-7B-Instruct (\texttt{q25}).
        \item 4 RCA datasets: Nezha, OpenRCA-500, RCAEval, lincyaw/rca.
        \item Training schedules: base, curriculum chain, and joint mix.
        \item Evaluation matrix: RCA metrics + general capability benchmarks.
      \end{itemize}
    \item Project role: this is not the final solution; it is the initialization phase that maps the SFT capability boundary.
  \end{itemize}
\end{frame}

\begin{frame}{SFT Wins: The Model Can Learn RCA Signals}
  \begin{itemize}
    \item $q25_{mix}$ became the strongest complete row by \texttt{recommendation\_f1}:
      \begin{itemize}
        \item OpenRCA-500: 0.305
        \item RCAEval: 0.733
        \item lincyaw: 0.242
      \end{itemize}
    \item After base-mismatch re-eval, $q25_{curr\_4\_lincyaw}$ showed real cumulative curriculum signal:
      \begin{itemize}
        \item lincyaw \texttt{service\_set\_f1}: 0.272 pre-fix $\rightarrow$ 0.563 post-fix.
        \item lincyaw \texttt{primary\_service\_match}: 0.454 $\rightarrow$ 0.723.
        \item RCAEval \texttt{severity\_match}: 0.041 $\rightarrow$ 0.818.
      \end{itemize}
    \item Takeaway: SFT is useful for injecting RCA structure, service identification, and recommendation behavior.
  \end{itemize}
\end{frame}

\begin{frame}{SFT Boundaries: Not Always a Win}
  \begin{itemize}
    \item General benchmark regression appears after direct RCA SFT:
      \begin{itemize}
        \item $q25_{mix}$ GSM8K drops 0.795 $\rightarrow$ 0.715.
        \item $q25_{mix}$ TruthfulQA drops 0.647 $\rightarrow$ 0.589.
      \end{itemize}
    \item Takeaway: SFT can teach RCA behavior, but general-capability regression must be tracked before scaling.
  \end{itemize}
\end{frame}

\begin{frame}{Why Wrapper Memory Follows From RCA-Demo}
  \begin{itemize}
    \item Direct base-model SFT exposes three project risks:
      \begin{enumerate}
        \item capability drift without the original pretraining distribution;
        \item output-format fragility under LoRA updates;
        \item evidence-grounding loss even when JSON surface form improves.
      \end{enumerate}
    \item Frozen base + wrapper separates concerns:
      \begin{itemize}
        \item base model keeps general reasoning and coding capability;
        \item wrapper absorbs long context, tool observations, and RCA/debug traces;
        \item failures can be isolated to memory compression instead of the whole model.
      \end{itemize}
    \item This turns \texttt{rca-demo} from a final SFT path into evidence for the RCA foundation-model architecture choice.
  \end{itemize}
\end{frame}

\begin{frame}{Core Architecture}
  \begin{columns}[T]
    \begin{column}{0.52\textwidth}
      \begin{block}{Memory update}
        \[
          m_t = G_\phi(m_{t-1}, c_t)
        \]
        \[
          y_t = F_\theta(q_t, m_t)
        \]
      \end{block}
      \begin{itemize}
        \item $F_\theta$: frozen 7B/8B base model.
        \item $G_\phi$: trainable wrapper.
        \item $m_t$: compact memory state.
        \item $c_t$: new context chunk or tool observation.
      \end{itemize}
    \end{column}
    \begin{column}{0.44\textwidth}
      \begin{block}{V0 memory type}
        Start with explicit memory tokens.
      \end{block}
      \begin{itemize}
        \item Easy to train and ablate.
        \item Compatible with frozen decoder models.
        \item Safer than writing into weights.
        \item Hidden-state memory and LoRA memory are later ablations.
      \end{itemize}
    \end{column}
  \end{columns}
\end{frame}

\begin{frame}{Why Freeze the 8B Base?}
  \begin{itemize}
    \item We start around 8B because it is strong enough for reasoning and still feasible for repeated experiments.
    \item Candidate bases:
      \begin{itemize}
        \item Llama-3.1-8B-Instruct as a secondary validation base.
        \item Qwen2.5-Coder-7B or similar for public debug-trace extension.
      \end{itemize}
    \item The base remains frozen in v0.
    \item Training only the wrapper limits catastrophic forgetting and makes the contribution easier to isolate.
  \end{itemize}
\end{frame}

\begin{frame}{Evaluation Modes}
  \begin{block}{Direct prediction}
    Input: issue text, code context, stack trace, logs, failing test output.\\
    Output: root cause, evidence, fix recommendation.
  \end{block}

  \begin{block}{Agentic RCA}
    The model can call tools such as search, open file, run tests, and inspect traces.\\
    Test-time training updates the wrapper from tool observations; the frozen base does not change.
  \end{block}

  \begin{block}{Baseline comparison}
    Full context vs. summary vs. retrieval vs. learned memory.
  \end{block}
\end{frame}
