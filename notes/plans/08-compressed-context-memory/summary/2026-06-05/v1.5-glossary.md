## Gate signals (the gate's inputs) — "which feature, what it means"
Computed per item from a forward pass; full codebook in [`v1.5-metric-candidates-2026-06-04.md §3/§7`](../2026-06-04/v1.5-metric-candidates-2026-06-04.md). **General** = same direction in all 7 families (gate-worthy); **model-specific** = do not transfer.

| signal | plain meaning | formula sketch | verdict |
|---|---|---|---|
| **`delta_last`** | size of the **last** memory-write step | $\|\Delta m_T\|$ | ✅ **general — the gate** (AUROC 0.59–0.80) |
| `delta_mean`/`delta_trend` | avg / trend of write-step sizes | $\overline{\|\Delta m_t\|}$ ; $(\|\Delta m_T\|-\|\Delta m_1\|)/T$ | ✅ general (≈`delta_last`) |
| `slot_norm_mean`/`_max` | how big the memory tokens are | $\overline{\|m_k\|}$, $\max_k\|m_k\|$ | ✅ general |
| `drift_mem` | total memory movement | $\|m_T-m_0\|$ | ✅ general |
| `mem_influence_span` | causal effect of memory on the answer span | $\overline{\mathrm{KL}(p\,\|\,p_{\text{mem-ablated}})}$ over span | ✅ general (1 new candidate that transfers) |
| `top5_mass_w`,`conf_w` | wrapper output confidence | $\sum\text{top-5 }p$, $\max p$ | ⚠️ partial (weak on GLM/14B) |
| `ll_conf_w_l`,`ll_margin_w_l` | logit-lens confidence/margin per layer | softmax of `unembed(norm(h_l))` | ❌ **model-specific** (single-model #1 → ≈chance on GLM) |

## How to read the numbers (high-level; math → `settings.md`)
- **AUROC(signal, label).** *"Take a random item where the wrapper helped and one where it didn't — how often is the signal larger on the helped one?"* **0.5 = no info (chance)**; **>0.5** = higher signal ⇒ more likely to help (positive predictor); **<0.5** = inverse. **|AUROC−0.5| = strength.** Threshold-free + robust to class imbalance ⇒ the right tool for "does this signal predict usefulness?". *Why it backs the claim:* `delta_last` 0.59–0.80 in **all 7** ⇒ in every model a bigger last memory-write ranks helpful inputs above harmful ones ⇒ a forward-pass signal can gate the wrapper **model-agnostically**.
- **LOMO AUROC.** Same metric, gate **trained on 6 families, tested on the 7th unseen** → 0.71 ⇒ generalizes to a new base with **no tuning** (the deployment claim).
- **Δ = native\_w − native\_0.** Actual benchmark change from using the wrapper vs base alone. **>0 helps, <0 hurts**; boundary = mean Δ & %help by distance.
- **do-no-harm "32/35".** Of 35 (model×bench) cells, gated ≥ base on 32 ⇒ the gate rarely lets the wrapper hurt.
