## 0. Why these defaults (grounded in our verified facts)
- F1/F2: compression preserves **meaning**, destroys **exact identifiers** (quality_hard 0.48 vs full 0.13; RULER 0.00 + collapse). F4: raw recent window wins when evidence is recent. ⇒ **neither alone is enough → hybrid.**
- F8: transfer is driven by hidden stats, not surface similarity ⇒ **generalization must be trained + measured intrinsically.**
- F3: held-out gate is only net-positive when full is degraded ⇒ **do-no-harm must be a conformal contract, not "beats full".**

## 1. The method — **Hybrid Evidence Memory + conformal router** (HEM)
Reader input per query:  `[ sink ; ctx[-W:] (raw recent) ; M (compressed over ALL ctx) ; query ]`
- **M** = our self-compressor over the full (over-window, recurrent) context → K soft tokens (global/spread evidence).
- **raw window W** = last W tokens verbatim (local detail + exact identifiers → fixes RULER/bfcl_multiple copy failure).
- **conformal router** chooses the cheapest sufficient path per query among {base, window, M, window+M, full}, with τ calibrated on a held-out split for a target risk (do-no-harm with coverage).
- **domain-invariant training**: encoder trained with a domain-adversarial / invariance penalty on M so the compressed representation transfers across domains.
- Base stays **frozen** (Gated-DeltaNet hybrid → soft-prompt/M is the natural interface); only mem + proj + read-LoRA train.

**Knobs (new):** `GCM_COMBINE=1`, `GCM_COMBINE_WIN=W` (raw-window len), `GCM_INVARIANCE` (λ for domain-adv head), router = conformal `_gate_cv` extension.

## 2. Claims → experiments
- **C1 Necessity.** HEM ≫ feasible no-compression baselines on over-window/semantic-spread tasks.
  - Benches: **quality / quality_hard / narrativeqa-OW / ∞Bench-reasoning / NoCha / locomo** (semantic) + **RULER length-sweep** (reported as the *retrieval* honest-negative where raw-window wins).
  - Baselines: truncated-full · TXL/StreamingLLM(W) · SFT-LoRA(full-cost) · Gist · Cartridge · RAG.
  - Figure: accuracy vs context-length; crossover where HEM overtakes truncation/window.
- **C2 Domain generalization.** invariant-M keeps compress-quality cross-domain; intrinsic signatures predict transfer.
  - **5×5 transfer matrix** (train domain A → eval B): {wiki-QA, narrative, tool, ops, web/dialogue}; report in/cross-task/cross-domain; with vs without invariance loss.
  - **Intrinsic probes** (from `v2.0.0-plan.md §5`, down-select to 4–6): M↔full CKA, hidden-OOD score, attention-mass-on-M, headroom + exact-identifier split, gate-ECE. Correlate probe ↔ transfer drop.
- **C3 Conformal do-no-harm.** held-out/conformal router → quality ≥ feasible baseline at stated coverage; rescues when full degraded.
  - Extend `_gate_cv` → conformal τ; report risk-coverage + cost-coverage Pareto (3-way held-out).

## 3. Ablations (HEM is the new axis; method must be robust)
- window W ∈ {0(=pure M), 256, 1024, sink+1020}; M K ∈ {128, 256}; M-only vs window-only vs hybrid.
- invariance λ ∈ {0, small, large}; adversarial vs IRM head.
- router: 2-way vs 3-way; in-sample vs held-out vs conformal.
- (Tier-2 if time) copy-slots on/off (exact-string fix); compress-then-retrieve on/off (RULER/scaling fix).
