# Context distillation

> A self-distillation technique where a **teacher** (the same LLM, *with* a relevant context in its prompt) generates targets that train a **student** (the same LLM, *without* that context) to behave identically. The context is thereby moved from the prompt into the model's parameters.

## Structure

**Contained by**: *knowledge distillation* (broader; not a folder here yet — self-distillation is a special case).

**Contains**:
- **Query-dependent CD** — distill the teacher only on a single test query (Eq. 1 in D2L paper). Cheap but overfits.
- **Query-independent CD** — distill over many synthetic queries generated from the context (Eq. 2 in D2L paper). Robust but expensive.
- **Amortized / meta-learned CD** — train a hypernet to produce the CD result in one forward pass (Doc-to-LoRA).
- **Prefix-tuning-style CD** — internalize context into prefix tokens rather than weights (Cartridges, Gisting).
- **Generative-adapter style CD** — train hypernet with next-token-prediction loss on ground-truth answers instead of KL on teacher distribution (Generative Adapter).

## Nearest neighbors

- [`inference-time-training`](../inference-time-training/) — CD is the canonical objective for "internalize context as weights."
- [`hypernetworks`](../hypernetworks/) — amortized CD requires a hypernet.
- [`long-context`](../long-context/) — CD is one of the two main strategies (besides architectural fixes) for going past the model's native context window.
- [`lora-peft`](../lora-peft/) — modern CD always outputs a LoRA, not a full fine-tune.

## Key concepts

- **Self-distillation** — teacher and student share the same base parameters; only the *information available to each* differs. Distinguishes CD from classical knowledge distillation (different teacher / student models).
- **KL on teacher distribution vs NTP on gold tokens** — Doc-to-LoRA shows KL beats NTP empirically (richer supervision). Generative Adapter uses NTP and pays for it on ROUGE-recall.
- **Synthetic query generation** — the expensive part of standard CD. Typically 10–25 queries per context, generated by a stronger model.
- **Internalization** — successful CD means the student answers as if it had the context, even though the context is absent at inference. The internalization is, in effect, a re-encoding of the context into weight space.
- **Interference** — CD-trained LoRAs assume subsequent queries relate to the internalized content. If they don't (Table 8 in D2L), the LoRA can degrade base performance. Active research problem.

## Open questions / live debates

- **Does CD scale to long-horizon multi-turn conversations** (vs static documents)?
- **Composition** — when many separate CDs are needed (one per user, per session, per document), do their LoRAs interfere when combined?
- **Forgetting** — what's the right unlearn mechanism for CD'd information?
- **CD vs RAG** — when does internalization beat retrieval, and vice versa? Probably depends on hit rate and update frequency, but the curve isn't well characterized.
